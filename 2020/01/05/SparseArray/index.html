SparseArray

  • SparseArray 是 Android 提供的轻量级数据结构，从字面意思看指的是稀疏数组
  • 因为 SparseArray 不需要对 key 和 value 进行 auto-boxing，所以在某些情况下要比 HashMap 的性能更好
  • SparseArray 主要使用两个一维数组来分别存储 key 和 value，不需要额外的数据结构

稀疏数组

  • 稀疏数组可以看做是普通数组（值的无效数据量远大于有效数据量）的压缩
  • 使用额外的行列来对二维数组中每个值进行记录，从而缩小数组的规模

实现

创建

  • SparseArray 提供了两个构造函数，默认的初始大小为10，也可以指定数组大小

插入

  • 插入数据时，先使用二分查找在 mKeys 数组中查找是否存在相同的 key

  • 如果 i >= 0 说明存在相同的 key，直接将 mValues 数组中对应角标的值替换掉

  • 如果 i < 0 说明 key 不存在，将 i 取反获得要插入的角标位置

  • 插入的位置没有越界，并且值已被替换为 DELETED，新的值可以直接进行插入

  • 如果标记需要回收，并且 mSize >= mKeys.length 则触发 gc() 方法并重新进行二分查找对角标进行定位

查找

  • 使用二分查找进行查找，如果查找结果为0或者目标的值被标记为 DELETED 则代表 key 无效

删除

  • 通过二分查找找到 key 对应的索引，将 value 替换为 DELETED

  • 回收标识置为 true

gc()

  • 实际上是压缩存储，删除一些被 DELETE 标记的元素

总结

  • SparseArray 避免了自动装箱，数据结构不依赖于外部对象映射

    HashMap 是通过 Hash 算法来决定每个元素的存储位置，存放的都是数组元素的引用，通过每个对象的 Hash 值来映射对象

  • SparseArray 使用二分查找，添加删除需要在数组中执行，效率相对较慢，但不需要额外内存空间存储外部映射，节省内存
  • 由于底层基于数组拷贝实现插入操作，因此数据量较大的情况下 SparseArray 性能较差
  • 特别注意 SparseArray 对于高并发情况，并不是线程安全的
