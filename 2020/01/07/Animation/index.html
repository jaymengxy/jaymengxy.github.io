Animation

Animation

View Animation

  • 视图动画，也叫补间动画，可以在一个视图容器内执行一系列的简单变换（位置、大小、旋转、透明度）
  • 可以通过 xml 或者 Android 代码定义，建议使用 xml 定义，更具有可读性和可重用性
  • 常用的有 AlphaAnimation RotateAnimation ScaleAnimation TranslateAnimation 和 AnimationSet，对 set 标签使用 Animation 属性时会对该标签下所有子控件都产生影响
  • 补间动画执行后并未改变 View 的真实布局属性值
  • 可以通过继承 BaseInterpolator 实现 NativeInterpolatorFactory 接口或创建 xml 自定义插值器

Drawable Animation

  • 在 xml 创建根节点 <animation-list> 包含每一帧动画的 <item>
  • 使用 ImageView 的 setBackgroundResource() 加载动画 xml
  • 通过 getBackground() 获取 AnimationDrawable 对象，调用 start() 开启动画
  • start() 方法不能在 Activity 的 onCreate() 方法中调用，因为 AnimationDrawable 对象还未完全附着到 window 上，最好是在 onWindowFocusChanged() 时调用

Property Animation

  • 属性动画，是通过修改属性值实现的

    类名 xml关键字
    ValueAnimator <animator> 在一个特定的时间里执行动画
    TimeAnimator 不支持 时序监听回调工具
    ObjectAnimator <objectAnimator> 一个对象的一个属性动画
    AnimatorSet <set> 动画集合

ObjectAnimator

  • 继承自 ValueAnimator 允许置顶要进行动画的对象以及该对象的一个属性，需要目标对象的属性提供指定的处理方法（get()/set()方法）
  • ObjectAnimator 提供了 ofInt ofFloat ofObject 这三个常用的方法用来设置动画作用的元素、属性、开始、结束等任意属性值，当属性只设置一个时，就把通过 getXX() 方法获取的值作为起点，设置的值作为终点。如果设置两个参数，那么一个是开始一个是结束。
  • ObjectAnimator 原理是不停的调用 setXX() 方法更新属性，所以 Object 必须声明有 getXX() setXX() 方法
  • 还可以通过 Listener 获取当前的属性值

PropertyValuesHolder

  • 多属性动画同时工作管理类，可以同时修改多个属性
  • ofPropertyValuesHolder

ValueAnimator

  • 属性动画中的时间驱动，管理着动画时间的开始、结束属性值和相应时间属性值的计算方法等
  • ValueAnimator 只是计算动画管理驱动，设置了作用目标 setTarger(View view) 但没有设置属性，需要通过 updateListener() 里设置属性才生效
  • 可以通过动画的计算去修改任何值

AnimatorSet

  • 把多个动画组合在一起，可以设置动画的时序关系，例如：同时播放 playTogether()、顺序播放 play().after()

Evaluators相关

  • 通过 Animator 提供的动画初始和结束值去计算一个动画的属性值
    • IntEvaluator 整数属性值
    • FloatEvaluator 浮点数属性值
    • ArgbEvaluator 十六进制颜色属性值
    • TypeEvaluator 自定义属性值接口

Interpolators

  • 插值器

ViewPropertyAnimator

  • 通过 View 的 animate() 方法可以的到一个 ViewPropertyAnimator 的属性动画
  • 可以直接是用一个 Animator 对象，对 View 的部分属性设置动画，会管理多个属性的 invalidate() 方法统一调用

LayoutAnimator

  • ViewGroup 中 View 添加时的动画，可以通过 LayoutTransition 对 ViewGroup 中的 View 进行动画设置展示
  • 使用 android:animateLayoutChanges="true" 允许使用系统默认的 ViewGroup 的 LayoutTransition 动画
  • 也可以创建 LayoutTransition 实例，传入 mViewGroup.setLayoutTransition() 方法并通过 LayoutTransition 实例的 setAnimator() 将其他属性动画添加设置到 LayoutTransition 内置动画类型之上

View Animation 和 Property Animation 的区别

  • View Animation 只能为 View 添加动画，支持种类少，改变的是 View 的绘制效果，View 的属性值没有改变，位置与大小也不会改变，代码少
  • Property Animation 可以为一个对象的任意属性添加动画，对象属性真实改变，属性改变时会自动刷新屏幕，不止用于 View，可以用于任何对象

Transition

SceneTransition 场景过渡动画

  • 以动画的形式实现两个场景的切换，在切换过程中通过 transition 来设置不同的过渡动画效果

Activity 过渡动画

  • API21以前通过 style 主题中统一设置或代码中添加 overridePendingTransition() 函数单独设置
  • API21以后可以使用 ActivityOptions + Transition 来实现
  • 可以使用 ActivityOptionsCompat 替换 ActivityOption
  • 允许使用 Transition 要么在 style 中设置 <item name="android:windowContentTransition">true</item>
  • 要么通过代码设置 getWindow().requestFeature(Window.FEATURE_CONTENT_TRANSITION)

SharedElementTransition 共享元素过渡动画

  • 共享元素的连接点是所有共享元素 View 的 transition name
  • 可以在 layout 中设置 android:transitionName 或 View.setTransition()
  • 可以通过代码或主题文件来设置
